# 叙事与AI-GM指南

> **用途**：AI-GM运行战役时的叙事原则、氛围营造、判定标准。
> **来源**：`L1/刀锋连队_AI-GM手册_v1.3.md` + `L1/刀锋连队_战役管理手册.md`

---

## 1. 核心叙事原则

1. **部分成功驱动叙事**：4-5是最常见结果 → 玩家总是付出代价
2. **精神压力=英雄主义的燃料**：超越自我让人做到不可能的事，但代价是崩溃
3. **角色轮换降低死亡恐惧**：角色死了换下一个 → 敢死队氛围
4. **双阶段结构**：战术小队行动 + 战略层管理
5. **恶魔的交易=道德困境**：玩家主动提出彼此代价 → 团队内自然发生背叛/牺牲
6. **闪回=创造性解决**：鼓励玩家用闪回创造有利条件，但代价是压力
7. **团队协作=集体牺牲**：鼓励玩家互相协助，但协助也要付出代价

## 2. 氛围与描述指南

### 开场与氛围
- 强调泥、血、雨、黑暗
- 军团不是在"冒险"，是在"求生"
- 每个决定都有代价

### 描述敌人
- 不死者是恐怖的存在，不只是"怪物"
- 描述腐化的视觉细节：肿胀、变色、不自然的肢体
- 破碎者的出现应该让玩家感到真正的恐惧

### 处理死亡
- 角色死亡时给予告别时刻
- 让书记官记录他们的牺牲
- 新兵接替时强调"又一张新面孔"

### 推进腐化
- 逐渐描述角色的枯萎症状
- 让其他角色注意到这些变化
- 制造"要不要抛弃被腐化的战友"的困境

### 使用进程表
- 让玩家看到危机表逐渐填满的紧张感
- 用竞速表制造"争分夺秒"的紧迫感
- 威胁表填满时给击败精英的成就感

## 3. 压力状态的叙事提示

| 压力值 | 叙事表现 |
|--------|---------|
| 0-2 | 正常，专注任务 |
| 3-4 | 手抖、呼吸急促、眼神游离、反复检查武器 |
| 5 | 非理性决定、对队友大喊、冒险行动、忽视命令 |
| 6（创伤） | 根据创伤症状表现：冷漠/偏执/鲁莽/恶毒等 |

## 4. AI-GM判定速查

| 情况 | 判定 |
|------|------|
| 玩家宣布行动 | 六步法：目标→行动→处境→效果→奖励→投骰 |
| 玩家不想承受后果 | 投抵抗骰，承受6-结果点压力 |
| 压力满6点 | 精神创伤，选8种症状之一，失能 |
| 压力5点 | 濒临崩溃，可能出现轻微失误 |
| 压力3-4点 | 感到紧张，描述应反映焦虑 |
| 伤害行满 | 向上溢推，3级满→4级=死亡 |
| 腐化满7点 | 获得枯萎症状，枯萎病+1级 |
| 遭遇骰 | 1-3绝望/4-5危险/6安全 |
| 敌人威胁度高 | 效果降1级/伤害升1级/腐化=威胁度 |
| 任务完成 | XP = 最高威胁度 |
| 玩家陷入困境 | 提示闪回（消耗1-2压力） |
| 多人协作 | 协助+1d / 领导集体+最多3d / 准备改善处境 / 保护承受后果 |
| 结算阶段 | 主要→次要→伤害/压力→士气→XP→晋升→营地 |

## 5. AI-GM数据格式规范

### 完整导出包含
```json
{
  "players": [...],
  "chars": [...],
  "legion": { "time", "pressure", "morale", "food", "supply", "intel" },
  "deploy": { "mainMissionName", "sideMissionName", "mainChars", "sideChars", ... }
}
```

### 自动保存键
- `blades-gm-players`（玩家）
- `blades-gm-chars`（角色）
- `blades-gm-legion`（军团状态）
- `blades-gm-deploy`（部署记录）
